# 仓位建议生成算法说明

> 主题：从「筛选出的一组基金」出发，如何**穿透前十大持仓 → 行业暴露聚类 → 基于相关性再分配 → 生成每个赛道的目标仓位**。
>
> 涉及代码：
> - 聚类：`backend/app/cluster/algo/`（`dedup` · `matrix` · `similarity` · `clustering` · `scoring` · `labeling` · `pipeline`）
> - 仓位：`backend/app/position/algo/`（`pipeline` · `optimize` · `prosperity` · `deviation` · `recommend`）

---

## 0. 整体链路

一个「预设」保存了一组筛选条件，把某一时点的筛选结果固化成**镜像快照**——一组基金，每只都带着它的「前十大持仓」和「收益/回撤/夏普/规模」等指标。仓位建议就建立在这份快照之上：

```
预设镜像快照（一组基金 + 各自前十大持仓 + 指标）
   │
   │  ① 份额去重          A/C/E 等同一只基金的多份额只留一只          dedup.py
   ▼
   │  ② 矩阵构建          前十大持仓 → 基金×股票矩阵 + 基金×行业矩阵     matrix.py
   ▼
   │  ③ 相似度 / 距离      Bray-Curtis 重叠度，0.6·股票 + 0.4·行业       similarity.py
   ▼
   │  ④ Ward 层次聚类      把口味相近的基金聚成「赛道」（簇）            clustering.py
   ▼
   │  ⑤ 簇组装 + 综合分    簇名 / 特征股 / 行业暴露 + 簇内排序           labeling.py · scoring.py
   ▼
   │  ⑥ 选代表基金         每簇从 TOP5 候选里挑 1 只（行业去重降相关）    optimize.select_funds
   ▼
   │  ⑦ 权重再分配         等权基准 → 按行业拥挤度做最小再分配（cap 上限）  optimize.rebalance_weights
   ▼
   每个赛道一只代表基金 + 目标占比（∑=100%）+ 加码/标配/减码 + 组合穿透/回测
```

**一句话抓住设计取向**：聚类负责「把会同涨同跌的基金归到一个赛道，避免在同一处重复下注」；仓位则**默认回归等权**，只为了「压低赛道之间的底层行业相关性」而对等权做最小幅度的微调。景气度、乖离度这些择时信号**算出来只作展示，不参与调权**（原因见 §7）。

---

## 1. 输入：筛选 + 前十大持仓

聚类的输入不是基金的净值，而是它的**持仓结构**。每只基金携带的 `holdings` 里，只有 `holding_type == "stock"` 且同时有股票代码与持仓占比（`hold_ratio`）的条目参与建模（见 `matrix._stock_holdings`）。

> 因此：**没有股票持仓数据的基金（如纯债、库内无穿透的指数基金）不会进入聚类的相似度计算**，它们被单独收进一个「其他」簇（`cluster_id=0`），不丢弃也不参与配权（`pipeline.run` + `labeling.other_cluster`）。

---

## 2. ① 份额去重（`dedup.py`）

同一只基金的 A / C / E / D 等份额，持仓 100% 相同、净值只因费率微差。若都进分析池，会被当成多只独立基金，**虚高分散度、污染聚类**。去重规则（宁可漏合并、不可错合并，两层）：

1. **同名分组**：把基金名末尾的份额字母（及「(前端)/(后端)」后缀）去掉，得到「基金主体名」，同名归一组。
2. **持仓确认**：组内再按前十大股票的重叠度确认——交集 / 较小集 ≥ `0.7`（`_OVERLAP_MIN`）才真正合并，避免「去掉字母后偶然同名」的两只不同基金被错并。纯债无股票持仓时则仅凭同名合并。

每个真份额子组只保留一只，规则确定可复现：按 `(sharpe_3y 降序, code 升序)` 取第一只（`_keep_first`）。

---

## 3. ② 矩阵构建（`matrix.py`）

把去重后的基金穿透到底层，构造两张矩阵（`FundMatrix`）。设有 `n` 只可聚类基金：

- **基金 × 股票矩阵** `stk_matrix`（`n × 股票数`）：第 `i` 行第 `k` 列 = 基金 `i` 持有股票 `k` 的**占净值比例** `hold_ratio`，没持有则为 0。
- **基金 × 行业矩阵** `ind_matrix`（`n × 行业数`）：把每只股票按**申万三级行业**（`industry_crud.label_of`，无映射兜底「其他」）归类，同一行业的持仓占比**累加**，得到基金在各行业上的暴露。

两张矩阵分别刻画「具体买了哪些股票」（细粒度）和「押在哪些行业」（粗粒度）。无任何股票持仓的基金在这一步被剔除（记入 `dropped_codes`，后续进「其他」簇）。

---

## 4. ③ 相似度与距离（`similarity.py`）

衡量「两只基金有多像」用 **Bray-Curtis 重叠度**，对股票层与行业层各算一次再融合。

**第一步：行归一**。每一行除以行和，使每只基金的暴露向量加总为 1（`_row_normalize`）——这样比较的是「结构相似」而非「仓位高低」。

**第二步：重叠度**。两只基金 `a`、`b` 在归一向量上的相似度是**逐维取小后求和**：

```
sim(a, b) = Σ_k min(a_k, b_k)          ∈ [0, 1]
```

含义直观：两个分布在每一维上「共同覆盖」的面积。完全相同的持仓 = 1，完全不重叠 = 0。向量化实现见 `_bray_curtis`：`np.minimum(mat[:,None,:], mat[None,:,:]).sum(axis=2)`。

**第三步：融合**。股票层与行业层加权融合（`W_STK=0.6`、`W_IND=0.4`）——以具体个股重叠为主、行业重叠为辅：

```
sim = 0.6 · sim_股票 + 0.4 · sim_行业
```

**第四步：转距离**。聚类需要「距离」，把相似度线性映射到 `[0, 2]`：

```
dist = 1 − (2·sim − 1) = 2·(1 − sim)
```

sim=1（相同）→ dist=0；sim=0（无关）→ dist=2。最后对称化并把对角线置 0。

---

## 5. ④ Ward 层次聚类（`clustering.py`）

用**纯 numpy 实现的 Ward 凝聚层次聚类**（`fcluster_ward`），等价于 `scipy.linkage(method='ward') + fcluster(criterion='maxclust')`。不依赖 scipy，是为了保持后端 pylint 满分、依赖最小；`n < 200` 时 O(n³) 完全够用。

- **凝聚过程**：每次合并当前距离最小的两个簇，用 **Lance-Williams 公式**更新合并后簇到其它簇的距离：

  ```
  d(新, k) = √( ((nᵢ+nₖ)·d(i,k)² + (nⱼ+nₖ)·d(j,k)² − nₖ·d(i,j)²) / (nᵢ+nⱼ+nₖ) )
  ```

  （Ward 准则：每步合并使**簇内方差增量最小**，倾向得到大小均衡、内部紧致的簇。）

- **目标簇数**（`pipeline._target_clusters`）：`clip(n // 10, 8, 40)`——大约每 10 只基金 1 个赛道，下限 8、上限 40。可聚类基金不足 3 只（`MIN_FUNDS`）则不出结果。

---

## 6. ⑤ 簇组装与簇内排序（`labeling.py` + `scoring.py`）

聚类只给出「谁和谁一簇」，`labeling.assemble` 把每个簇组装成可读的赛道：

- **簇名**：簇内 `ind_matrix` 行业均值 top3 标签拼接（如「半导体 / 消费电子 / 元件」）。
- **特征股**（`signature_stocks`）：簇内股票均值**减去全池基线**后差值最大的股票——凸显「这个赛道独有的偏好」，而非人人都买的大白马。
- **次要重叠股 / 实际资金暴露**：簇内被多只基金共同持有的股票、以及按重仓市值聚合的资金暴露榜（供分析展示）。
- **簇内基金排序**：按下面的「临时综合分」降序，排第一的就是这簇的 TOP1 候选。

**临时综合分**（`scoring.composite_scores`）——给簇内基金排座次用的近似质量分（完整年度排名评分缺数据，本轮用现有指标近似）：

```
score = 0.40·z(sharpe_3y) + 0.30·z(risk_return_ratio_3y) + 0.15·z(log 规模) + 0.15·z(年限)
```

其中 `z(·)` 是**以中位数为中心、标准差缩放**的标准分，缺失值取 0，并 clip 到 `[-2, 2]` 抗极端值。综合分在「可聚类 + 其他」全量池上统一计算，保证 z 口径一致。

---

## 7. ⑥⑦ 从赛道到仓位（`position/algo/pipeline.py`）

到这里每个赛道都有了一组按综合分排好序的候选基金。仓位生成做三件事：**选代表 → 定权重 → 出推荐**。

### 7.1 选代表基金（`optimize.select_funds`）

并非每簇硬取 TOP1。问题在于：聚类按行业聚，多个簇的主行业可能相同（例如都重仓「通信」），各取 TOP1 后穿透到底层高度重合、相关性强。于是每簇从综合分**前 5 名候选**（`TOPK=5`）里选 1 只，目标是：

```
maximize  平均质量(归一综合分)  −  12 · max(0, 组合最大单一行业占比 − cap)
```

即在「质量尽量高」的前提下，**只要簇内有更分散的次优基金，就用它把组合的单一行业穿透占比压到 cap 以下**（局部搜索，从全 TOP1 出发逐簇试替换）。选中的基金可能不是 TOP1，前端用 `cluster_rank`（1=TOP1，>1=为降相关选了次优）标注。

### 7.2 权重再分配（`optimize.rebalance_weights`）—— 本文标题里的「基于相关性生成仓位」

这是核心，也是最容易误解的一步。权重**以等权 1/N 为基准**，然后**只为了满足「单一行业穿透占比 ≤ cap」做最小幅度的再分配**：

```
目标 =  −12 · max(0, 最大单一行业占比 − cap)  −  4 · Σ(w − base)²
```

- 第一项是 **cap 硬上限**（用大惩罚松弛实现）：哪个行业穿透后占比超标，就要把权重从「贡献该拥挤行业」的簇挪走。
- 第二项是 **贴近等权的拉力**（`mu=4`）：行业不超限时，权重几乎原样保持等权（健康的温和分布）；超限时，移动**最少**的权重把它压回上限。

实现是「从一簇挪 `step=0.005` 给另一簇」的局部搜索，全程满足 `Σw=1`、`w ∈ [0.03, 0.25]`（`WMIN/WMAX`）。

> **为什么不按景气度调权？** 文件注释写得很直接：若再分配里去最大化景气，景气会被双重计入（综合分里已含夏普），且线性目标在盒约束下会产生**角点解**——权重全堆到少数高景气簇顶到 `WMAX`、其余压到 `WMIN`，两极分化。更关键的是 **walk-forward 回测显示「动量/景气调权」实测略跑输等权**。所以最终方案是：**回归等权，仅按行业相关性做最小再分配。**

### 7.3 景气度与乖离度（`prosperity.py` / `deviation.py`）—— 只展示，不调权

这两个信号仍然计算并随建议展示，作为「这个赛道当下热不热 / 是不是涨多了」的观察参考：

- **景气度**（0–100，四因子加权）：`momentum 动量(0.35)` + `risk_adj 风险调整(0.25)` + `breadth 广度(0.25)` + `consistency 一致性(0.15)`。动量按 1m/3m/6m 收益加权后跨簇 min-max 归一；风险调整用近 60 日 Sharpe-like 跨簇 rank 归一；广度看净值站上 MA20/MA60 的程度；一致性看最近连续正收益月数。数据不足的因子回退中性 50。
- **乖离度**（择时）：当前净值相对 MA20/MA60 的偏离百分比，`combined = 0.6·d20 + 0.4·d60`。

它们进入展示与文案，但**不进入 7.2 的权重计算**。

### 7.4 推荐标签（`recommend.py`）

加码 / 标配 / 减码**完全由「最终权重相对等权的偏离方向」决定**，也就是由行业拥挤度（相关性）决定，而非景气度：

| 条件 | 标签 | 含义 |
|---|---|---|
| `weight − base > +0.5%` | **加码** | 底层行业相对分散，降相关时增配 |
| `weight − base < −0.5%` | **减码** | 底层行业较拥挤，为降相关压低 |
| 否则 | **标配** | 行业拥挤度适中，等权标配 |

景气度只作为中文理由里的一句附注（「动量强度 XX，仅供参考」）。

### 7.5 均衡强度（cap）

`cap` 就是「单一行业穿透占比上限」，对应前端的均衡强度档位：

| 档位 | cap | 效果 |
|---|---|---|
| 松 | 0.22 | 允许行业更集中，权重更接近纯等权 |
| 中 | 0.18 | 居中 |
| 紧（默认） | 0.14 | 更分散，逼系统多选行业不重叠的代表基金、把拥挤赛道压得更低 |

cap 越紧，§7.1 越倾向选次优但分散的基金、§7.2 越会把权重从拥挤赛道挪走。

## 8. 产物：穿透、组合净值与回测

仓位定下后，`pipeline.run` 还会给出整组合的视角：

- **穿透（lookthrough）**：把各赛道代表基金的前十大持仓**按目标权重穿透累加**，得到「组合实际对每只股票 / 每个行业的暴露%」，并统计被 ≥2 只基金共同持有的**重叠股**（重叠越多说明底层越集中、代表基金越相关）。这与 §7 的 cap 约束口径一致——cap 控的就是这里的最大单一行业占比。
- **组合净值曲线**：各代表基金净值在共同起点 rebase 到 1.0，按目标权重加权合成组合净值，算出最大回撤、年化收益、年化波动、夏普（`_portfolio_curve` / `_portfolio_stats`）。
- **数据时效**：标注净值截止日（`nav_as_of`）与持仓季度（`holdings_quarter`），让使用者知道建议基于哪一期数据。

最终返回 `{items, portfolio, lookthrough, meta}`，`items` 按目标权重降序——每个赛道一行：代表基金、目标占比、加码/标配/减码、景气度与乖离观察值、迷你净值走势、前十大持仓。

---

## 9. 与实盘对账的衔接

上述「赛道 → 代表基金 → 目标占比」就是实盘对账（`reconcile`）的输入：把目标比例落到真实持仓上，按赛道对齐 + 金额化，算出「该买/该卖/该转多少钱」的操作指南。详见实盘对账模块。
